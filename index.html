Current support boundary

ขอบเขตการรองรับปัจจุบัน

  • Launcher route mode defaults to direct; cloudflare is explicit opt-in.
  • Checked Cloudflare-capable targets are claude-code, codex, zed, opencode, and hermes; openclaw can use Cloudflare only when compatibility is openai or anthropic.
  • gemini-cli stays direct-only in the current checked scope and falls back to direct when Cloudflare is requested.
  • opencode and hermes keep the same custom-provider root in both modes; the launcher only swaps which root/base gets injected.
  • Shell helper paths can apply changes where the checked script supports it.
  • PowerShell launcher paths are route-mode-aware and remain dry-run-first.
  • gemini-cli helper support is env/snippet/profile oriented.
  • hermes helper support is profile-local and writes .env, config.yaml, and SOUL.md for one dedicated Hermes profile path.
  • openclaw still requires the local openclaw command in PATH.
  • Checked local smoke coverage now verifies launcher route-mode behavior plus setup-script dry-run contracts in scope; this is not live provider, deploy, or hosted Pages proof.
  • route mode ของ launcher จะ default เป็น direct และ cloudflare เป็นการ opt-in แบบชัดเจน
  • target ที่ตรวจแล้วว่าใช้ Cloudflare ได้คือ claude-code, codex, zed, opencode และ hermes; ส่วน openclaw จะใช้ได้เมื่อ compatibility เป็น openai หรือ anthropic เท่านั้น
  • gemini-cli ยังเป็น direct-only ในขอบเขตที่ตรวจแล้วตอนนี้ และจะ fallback กลับไป direct เมื่อมีการขอ Cloudflare
  • opencode และ hermes จะใช้ custom-provider root ชุดเดิมทั้งสองโหมด โดย launcher มีหน้าที่สลับ root/base ที่ inject เข้าไปเท่านั้น
  • ฝั่ง shell สามารถ apply ได้ในจุดที่สคริปต์ที่ตรวจรองรับจริง
  • launcher ฝั่ง PowerShell รองรับ route mode แล้ว แต่ยังคงเป็นแนว dry-run-first
  • helper ของ gemini-cli เป็นแนวจัดการ env/snippet/profile
  • helper ของ hermes จะเขียน .env, config.yaml และ SOUL.md ให้ใน profile path เฉพาะของ Hermes
  • openclaw ยังต้องมีคำสั่ง openclaw อยู่ใน PATH
  • ตอนนี้มี smoke coverage แบบ local สำหรับ route-mode ของ launcher และ dry-run contract ของ setup scripts ในขอบเขตที่ตรวจแล้ว แต่ยังไม่ใช่การพิสูจน์แบบ live provider, deploy หรือ hosted Pages
