Current support boundary

ขอบเขตการรองรับปัจจุบัน

  • Launcher route mode defaults to direct; cloudflare is explicit opt-in.
  • Checked Cloudflare-capable targets are claude-code, gemini-cli, codex, zed, opencode, and hermes; openclaw can use Cloudflare only when compatibility is openai or anthropic.
  • gemini-cli now uses the protected Google / Gemini v1beta family when Cloudflare mode is requested, while direct mode keeps the native Gemini root.
  • opencode and hermes keep the same custom-provider root in both modes; the launcher only swaps which root/base gets injected.
  • Shell helper paths can apply changes where the checked script supports it.
  • PowerShell launcher paths are route-mode-aware and preview-first; direct PowerShell helpers remain the apply surface where that helper supports file writes or env/session output.
  • gemini-cli helper support is env/snippet/profile oriented.
  • hermes helper support is profile-local and writes .env, config.yaml, and SOUL.md for one dedicated Hermes profile path.
  • openclaw still requires the local openclaw command in PATH.
  • Checked local smoke coverage now verifies launcher route-mode behavior plus setup-script dry-run contracts in scope; this is not live provider, deploy, or hosted Pages proof.
  • route mode ของ launcher จะ default เป็น direct และ cloudflare เป็นการ opt-in แบบชัดเจน
  • target ที่ตรวจแล้วว่าใช้ Cloudflare ได้คือ claude-code, gemini-cli, codex, zed, opencode และ hermes; ส่วน openclaw จะใช้ได้เมื่อ compatibility เป็น openai หรือ anthropic เท่านั้น
  • gemini-cli จะใช้ protected Google / Gemini v1beta family เมื่อร้องขอ Cloudflare mode ส่วน direct mode จะคง native Gemini root ไว้
  • opencode และ hermes จะใช้ custom-provider root ชุดเดิมทั้งสองโหมด โดย launcher มีหน้าที่สลับ root/base ที่ inject เข้าไปเท่านั้น
  • ฝั่ง shell สามารถ apply ได้ในจุดที่สคริปต์ที่ตรวจรองรับจริง
  • launcher ฝั่ง PowerShell รองรับ route mode แล้ว แต่ยังคงเป็นแนว dry-run-first
  • helper ของ gemini-cli เป็นแนวจัดการ env/snippet/profile
  • helper ของ hermes จะเขียน .env, config.yaml และ SOUL.md ให้ใน profile path เฉพาะของ Hermes
  • openclaw ยังต้องมีคำสั่ง openclaw อยู่ใน PATH
  • ตอนนี้มี smoke coverage แบบ local สำหรับ route-mode ของ launcher และ dry-run contract ของ setup scripts ในขอบเขตที่ตรวจแล้ว แต่ยังไม่ใช่การพิสูจน์แบบ live provider, deploy หรือ hosted Pages

Remote launcher examples by tool

ตัวอย่าง remote launcher แยกตามเครื่องมือ

The per-tool rows below are one-shot commands. Use them when you want one command that starts PowerShell, downloads the launcher, and runs it immediately.

แถวแยกตามเครื่องมือด้านล่างคือคำสั่งแบบ one-shot ใช้เมื่อคุณต้องการคำสั่งเดียวที่เปิด PowerShell, ดาวน์โหลด launcher และรันทันที

If PowerShell is already open, use the separate session-based pattern underneath instead of pasting the one-shot command back into the same session.

ถ้าเปิด PowerShell อยู่แล้ว ให้ใช้ตัวอย่างแบบ session ด้านล่างแทน ไม่ต้องเอาคำสั่ง one-shot กลับมาวางซ้ำใน session เดิม

Inside an existing PowerShell session

ภายใน PowerShell session ที่เปิดอยู่แล้ว

Env

Env

  • ANTHROPIC_BASE_URL = direct/native service root for Claude Code, without appending /v1
  • ANTHROPIC_AUTH_TOKEN = your NodeClaw access key
  • ANTHROPIC_MODEL = optional default model override for the same shell/session
  • CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 = optional fallback when the gateway rejects Anthropic beta headers
  • DISABLE_INTERLEAVED_THINKING=1 = optional fallback when the provider path rejects interleaved thinking
  • ANTHROPIC_BASE_URL = service root แบบ direct/native สำหรับ Claude Code โดยไม่ต้อง append /v1
  • ANTHROPIC_AUTH_TOKEN = NodeClaw access key ของคุณ

Troubleshoot

Troubleshoot

  • If the first request fails immediately, verify the target still exposes /v1/messages and /v1/messages/count_tokens.
  • If the gateway rejects Anthropic beta headers/features, keep CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 and optionally DISABLE_INTERLEAVED_THINKING=1 before deeper debugging.
  • If ANTHROPIC_BASE_URL points to a non-first-party host, MCP tool search stays disabled unless that proxy forwards tool_reference blocks correctly.
  • ถ้า request แรก fail ทันที ให้ตรวจว่าปลายทางยังเปิด /v1/messages และ /v1/messages/count_tokens อยู่
  • ถ้า gateway ไม่รับ Anthropic beta headers/features ให้คง CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 และใช้ DISABLE_INTERLEAVED_THINKING=1 เพิ่มเมื่อจำเป็น ก่อนค่อย debug ต่อ
  • ถ้า ANTHROPIC_BASE_URL ชี้ไปที่ non-first-party host, MCP tool search จะยังถูกปิดอยู่ เว้นแต่ proxy นั้นจะ forward tool_reference blocks ได้ถูกต้อง
