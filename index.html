NodeClaw Helper Scripts

สคริปต์ตัวช่วยของ NodeClaw

Launcher-first helper distribution for NodeClaw

This release surface exposes the checked helper scripts that support the broader NodeClaw docs and tool-setup flow.

The broader product/docs domain basis is:

  • nodenetwork.ovh
  • nodenetwork.ovh/docs
  • nodenetwork.ovh/docs/tools

ศูนย์แจกจ่าย helper แบบ launcher-first สำหรับ NodeClaw

หน้านี้เปิดเผยสคริปต์ helper ที่ตรวจแล้ว ซึ่งรองรับ flow การตั้งค่าเครื่องมือและ docs ของ NodeClaw ในภาพรวม

โดเมนหลักของ product/docs คือ:

  • nodenetwork.ovh
  • nodenetwork.ovh/docs
  • nodenetwork.ovh/docs/tools

Supported tools

เครื่องมือที่รองรับ

claude-code gemini-cli codex openclaw opencode zed

Current support boundary

ขอบเขตการรองรับปัจจุบัน

  • Shell helper paths can apply changes where the checked script supports it.
  • PowerShell launcher paths remain scaffold-first and dry-run-only.
  • gemini-cli helper support is env/snippet/profile oriented.
  • openclaw still requires the local openclaw command in PATH.
  • ฝั่ง shell สามารถ apply ได้ในจุดที่สคริปต์ที่ตรวจรองรับจริง
  • launcher ฝั่ง PowerShell ยังเป็น scaffold-first และ dry-run-only
  • helper ของ gemini-cli เป็นแนวจัดการ env/snippet/profile
  • openclaw ยังต้องมีคำสั่ง openclaw อยู่ใน PATH

Quick start

Quick start

curl -fsSL https://darkwingtm.github.io/Nodeclaw-Helper/script/launcher.sh | bash -s -- wizard
curl -fsSL https://darkwingtm.github.io/Nodeclaw-Helper/script/launcher.sh | bash -s -- dry-run --tool gemini-cli
curl -fsSL https://darkwingtm.github.io/Nodeclaw-Helper/script/launcher.sh | bash -s -- apply --tool gemini-cli

Remote usage should call launcher only. Launcher fetches the helper payload it needs automatically.

การใช้งานแบบ remote ควรเรียก launcher อย่างเดียว แล้ว launcher จะไปดึง helper payload ที่ต้องใช้ให้อัตโนมัติ

powershell -ExecutionPolicy Bypass -c "irm https://darkwingtm.github.io/Nodeclaw-Helper/script/launcher.ps1 | iex"

Gemini CLI helper path

เส้นทาง helper ของ Gemini CLI

Primary setup story:

export GOOGLE_GEMINI_BASE_URL="https://payg.nodenetwork.ovh"
export GEMINI_API_KEY="<nodeclaw_access_key>"
gemini

Effective Google / Gemini-shaped route family still resolves under v1beta. The helper manages the same env-based path through a generated snippet/profile block.

เส้นทางหลักของการตั้งค่า:

export GOOGLE_GEMINI_BASE_URL="https://payg.nodenetwork.ovh"
export GEMINI_API_KEY="<nodeclaw_access_key>"
gemini

route family แบบ Google / Gemini จะวิ่งอยู่ใต้ v1beta ส่วน helper มีหน้าที่จัด env/snippet/profile สำหรับ path เดียวกันนี้
