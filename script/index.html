NodeClaw Helper Scripts

สคริปต์ตัวช่วยของ NodeClaw

Launcher-first helper distribution for NodeClaw

This release surface exists for the helper scripts that support the broader NodeClaw docs and tool setup flow.

The broader product/docs domain basis is:

  • nodenetwork.ovh
  • nodenetwork.ovh/docs
  • nodenetwork.ovh/docs/tools

ศูนย์แจกจ่าย helper แบบ launcher-first สำหรับ NodeClaw

หน้านี้มีไว้สำหรับสคริปต์ตัวช่วยที่รองรับ flow การตั้งค่าเครื่องมือและ docs ของ NodeClaw ในภาพรวม

โดเมนหลักของ product/docs คือ:

  • nodenetwork.ovh
  • nodenetwork.ovh/docs
  • nodenetwork.ovh/docs/tools

Supported tools

เครื่องมือที่รองรับ

claude-code codex openclaw opencode zed

Current support boundary

ขอบเขตการรองรับปัจจุบัน

  • Shell helper paths can apply changes where the checked script supports it.
  • PowerShell helper paths remain scaffold-first and dry-run-only.
  • Do not assume hosted curl | bash install is officially live.
  • openclaw requires the local openclaw command to already exist in PATH.
  • ฝั่ง shell สามารถ apply ได้ในจุดที่สคริปต์ที่ตรวจรองรับจริง
  • ฝั่ง PowerShell ยังเป็นแนว scaffold-first และ dry-run-only
  • อย่าตีความว่า hosted curl | bash เปิดใช้งานจริงแล้ว
  • openclaw ต้องมีคำสั่ง openclaw อยู่ใน PATH ก่อน

Recommended usage order

ลำดับการใช้งานที่แนะนำ

  • Start from the main NodeClaw docs surface when you want the full product context.
  • Use launcher.sh or launcher.ps1 as the generic helper entrypoint.
  • Run dry-run first.
  • Use direct per-tool scripts only when a more explicit tool-specific entrypoint is better.
  • เริ่มจากหน้า docs หลักของ NodeClaw ก่อน หากคุณต้องการ context ของ product ที่ครบกว่า
  • ใช้ launcher.sh หรือ launcher.ps1 เป็น generic helper entrypoint หลัก
  • รัน dry-run ก่อน
  • ใช้ direct per-tool scripts เมื่อคุณต้องการ entrypoint ที่เฉพาะเจาะจงกับเครื่องมือนั้นมากกว่า
