NodeClaw Helper Scripts

Launcher-first helper distribution for NodeClaw

This release surface exists for the helper scripts that support the broader NodeClaw docs and tool setup flow.

The real broader product/docs domain basis is:

  • nodenetwork.ovh
  • nodenetwork.ovh/docs
  • nodenetwork.ovh/docs/tools

Supported tools

claude-code codex openclaw opencode zed

Current support boundary

  • Shell helper paths can apply changes where the checked script supports it.
  • PowerShell helper paths remain scaffold-first and dry-run-only.
  • Do not assume hosted curl | bash install is officially live.
  • openclaw requires the local openclaw command to already exist in PATH.

Start here

$ bash ./launcher.sh list
Supported tools:

Launcher entrypoints

Use launcher as the main generic interface. It relies on sibling files in the same script/ directory, so use it from a repo checkout or a downloaded script folder.

launcher.shShell launcher launcher.ps1PowerShell launcher

Direct per-tool helpers

Tool-specific scripts remain available when a direct tool path is clearer than the generic launcher.

Linux / macOS shell helpers

Linux and macOS share the same direct shell helper files.

Recommended usage order

  • Start from the main NodeClaw docs surface when you want the full product context.
  • Use launcher.sh or launcher.ps1 as the generic helper entrypoint.
  • Run dry-run first.
  • Use direct per-tool scripts only when a more explicit tool-specific entrypoint is better.
